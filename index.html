📋 마스터 시트 + 백업 폴더 + Apps Script 설치 방법
  1. 새 구글 스프레드시트를 만들고 URL의 /d/.../edit 사이 시트 ID를 메모.
  2. 구글 드라이브에서 백업이 쌓일 폴더를 만들고 URL의 /folders/... 뒤 폴더 ID를 메모. 백업이 필요 없다면 이 단계는 건너뛰어도 됨.
  3. 1)의 시트에서 확장 프로그램 → Apps Script 클릭.
  4. 기본 Code.gs를 모두 지우고 guide-apps-script.js를 전체 복사해 붙여넣기.
  5. 상단의 MASTER_SHEET_ID에 시트 ID, BACKUP_FOLDER_ID에 폴더 ID를 채우고 저장. 백업을 안 쓰면 BACKUP_FOLDER_ID는 빈 문자열 ''로 두면 자동으로 백업 스킵.
  6. 함수 드롭다운에서 setupTest 선택 → ▶ 실행 → 권한 동의 → 실행 로그에서 ✓ 시트 연결 성공, ✓ 백업 폴더 연결 성공 확인.
  7. 배포 → 새 배포 → 웹 앱 (실행: 나 / 권한: 모든 사용자) → URL 복사.
    기존 배포가 있으면 배포 관리 → 연필 → 새 버전 → 배포 업데이트. URL이 유지됩니다.
  8. 위 Apps Script URL과 마스터 시트 URL에 각각 붙여넣기 → 상단 점이 초록이면 연결 완료.
💾 백업 폴더 & 이력 이어쓰기 (다른 컴퓨터·다른 계정)
  • 자동 백업 (로컬 다운로드) — 마스터 시트로 전송할 때마다 브라우저의 다운로드 폴더에 탭이름__HHmmss.json 파일이 자동으로 떨어집니다. 시트가 삭제되거나 손상돼도 이 JSON으로 복구 가능. Drive 권한 없이 작동.
    💡 Mac의 다운로드 폴더에 매번 쌓이니, 본인의 백업 폴더로 주기적으로 옮기시면 됩니다.
  • 가져오기 — 액션 바의 📥 백업 가져오기를 눌러 JSON 파일을 선택하면 미리보기·프로젝트명·섹션 칩이 한 번에 복원됩니다. 다른 컴퓨터로 옮길 때 가장 빠른 방법.
  • 다른 컴퓨터에서 이어 쓰기 (같은 내 계정) — 그 컴퓨터에서 index.html을 열고 Apps Script URL · 마스터 시트 URL 두 개만 채우면 같은 시트에 누적됩니다. 이전 입력 이력은 백업 폴더의 JSON을 하나 골라 가져오기로 복원.
  • 협업자(다른 계정) 합류 — 협업자에게:
    ① Apps Script URL 알려줌 (권한 "모든 사용자"라 URL만 알면 사용 가능)
    ② 마스터 시트를 공유 → 링크가 있는 모든 사용자 → 편집자(또는 뷰어)로 공유
    ③ 백업 폴더도 같은 방식으로 공유하면 둘이서 동일한 백업 보관 가능
  • ⚠ Apps Script는 "나로 실행" 모드라 누가 호출하든 하림의 권한으로 시트에 씁니다. 즉 협업자가 시트 권한이 없더라도 데이터는 정상 기록돼요. 단, 결과 카드의 "시트에서 열기" 버튼이 동작하려면 그 계정에 시트가 공유돼 있어야 함.
